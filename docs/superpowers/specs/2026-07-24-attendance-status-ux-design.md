# 출결 상태 확인 화면 UX 개선 설계

## 배경

현재 홈 화면은 사용자 정보와 오늘 예정된 동작 다음에 Google 인증 카드를 표시하고,
인증이 끝나면 그 아래에 오늘 출결 카드를 새로 추가한다. 사용자는 인증을 시작한
위치와 결과가 나타나는 위치가 달라 결과를 바로 찾기 어렵다. 오늘 출결 카드는 서버가
보낸 시각 문자열과 내부 네트워크 판정값을 그대로 보여주므로 사용자용 정보로도
다듬을 필요가 있다.

이번 변경은 Android 배포 화면을 우선 개선한다. 진행 중인 iOS Safari 콜백
프로토타입과 섞이지 않도록 `main`에서 분기한 별도 브랜치에서 작업하고, 나중에 iOS
브랜치가 `main`을 반영할 때 공용 화면 변경과 iOS 읽기 전용 표시를 함께 조정한다.

## 목표

- 인증 전 카드가 어떤 목적인지 제목만 보고 이해할 수 있게 한다.
- 인증 결과가 인증을 시작한 자리에서 바로 나타나게 한다.
- 날짜와 출결 시각을 사람이 읽기 쉬운 형식으로 표시한다.
- 실제 출결 반영이 서버 상태 재조회로 확인된 경우에만 명확한 완료 피드백을 제공한다.
- 기존 출결 전송 안전 원칙과 실패 후 재조회 흐름을 유지한다.

## 범위

### 포함

- 인증 카드와 오늘 출결 카드의 위치 및 상태 전환
- 인증 카드 문구
- 오늘 출결 날짜·요일 및 시각 표시
- 네트워크 판정 표시 방식
- 정보 갱신 및 출결 동작 완료 피드백
- 관련 컨트롤러·표시 도우미·위젯 테스트
- Android 릴리스 빌드와 연결된 실기기 업데이트 검증

### 제외

- 인증 방식, App Link, API 주소 또는 출결 허용 규칙 변경
- 알림 예약 알고리즘 변경
- 출결 정보의 디스크 저장
- iOS Safari 콜백 프로토타입 구현 변경

## 화면 구조

홈 화면의 카드 순서를 다음과 같이 고정한다.

```text
사용자 인사
사용자 정보
출결 정보 확인하기 또는 오늘 출결
오늘 예정된 동작
Google 인증 보안 안내
```

인증 전에는 `출결 정보 확인하기` 카드가 보인다. 기존 설명 메시지와 오류·재시도
상태는 이 카드 안에서 유지하고 기본 버튼은 `Google 인증 시작`으로 표시한다.

인증과 최초 상태 조회가 성공하면 인증 카드를 제거하고 정확히 같은 자리에 오늘 출결
카드를 표시한다. 별도 결과 카드를 화면 아래에 추가하지 않으며 강제 스크롤도 하지
않는다. 프로세스가 재시작되어 메모리 토큰과 상태가 사라지면 다시 인증 전 카드로
돌아간다.

## 오늘 출결 표시

카드 제목은 한국 날짜와 요일을 포함해 다음 형식으로 표시한다.

```text
오늘 출결 · 7월 24일(금)
```

출결 상태는 한 줄씩 세로로 나열하지 않고 2×2 타일로 표시한다. 첫째 행은 입실과
퇴실, 둘째 행은 외출과 복귀다. 값이 없으면 `없음`, 값이 있으면 `HH:mm` 형식으로
표시한다.

```text
입실              퇴실
09:01             없음

외출              복귀
12:34             없음
```

각 타일은 동작명을 작게, 시각을 더 크게 표시한다. 일반 휴대폰 폭에서는 2열을
사용하되 큰 접근성 글자 크기나 각 타일의 최소 너비를 확보할 수 없는 좁은 화면에서는
자동으로 1열로 전환한다.

서버가 ISO 8601 타임스탬프와 시간대를 보낸 경우 한국 시각으로 변환한다.
`HH:mm` 형태는 그대로 표시하고 `HH:mm:ss` 형태는 초를 제외한다. 예상하지 못한
문자열은 화면 오류를 일으키지 않도록 원문을 유지한다. 화면용 변환은 프레젠테이션
도우미에서 담당하고 API 응답 모델의 원본 값과 출결 판정 규칙은 변경하지 않는다.

`네트워크 허용: 예` 행은 제거한다. `networkAllowed`는 가능한 동작을 계산하는 내부
판정에 계속 사용하고, 값이 거짓일 때만 기존 경고 문구를 표시한다.

## 갱신 및 완료 피드백

인증 콜백이나 수동 상태 재조회로 새 스냅샷을 받은 경우 카드 제목 영역에
`방금 업데이트됨`을 약 5초간 표시한다. 단순 조회에서는 특정 출결 행을 강조하지
않는다.

출결 동작은 다음 순서를 모두 통과해야 성공 피드백을 낸다.

```text
출결 요청 전송
→ 오늘 출결 상태 재조회
→ 요청한 동작의 시각이 서버 상태에 존재하는지 확인
→ 스냅샷 반영
→ 완료 피드백
```

성공하면 해당 동작의 타일 전체를 성공 계열 배경색으로 약 5초간 강조하고 체크 아이콘과
`방금 처리됨`을 표시한다. 동시에 화면을 막지 않는 스낵바로
`입실이 완료되었습니다`처럼 안내한다. 모달 확인창은 추가하지 않는다.

동작 완료 이벤트에는 증가하는 식별값을 둔다. 화면은 마지막으로 처리한 식별값보다
새로운 이벤트에 대해서만 스낵바를 한 번 표시해 컨트롤러의 다른 상태 알림으로
중복 노출되지 않게 한다. 행 강조와 `방금 업데이트됨` 표시는 화면 상태와 타이머로
관리하고 화면이 폐기될 때 타이머도 해제한다.

출결 요청 이후 상태 재조회가 실패하면 성공 이벤트를 만들지 않는다. 기존 원칙대로
출결 요청을 다시 보내지 않고 상태만 재조회한다. 재조회 결과에서 원래 요청한 동작의
반영이 확인된 경우에만 그 시점에 완료 이벤트를 만들고 성공 피드백을 표시한다.

## 상태와 책임

### `AttendanceController`

- 최신 `AttendanceSnapshot`과 기존 오류·재시도 상태를 계속 소유한다.
- 성공적으로 확인된 마지막 출결 동작과 완료 이벤트 식별값을 노출한다.
- 상태 조회 성공 식별값을 증가시켜 화면이 새 스냅샷을 구분할 수 있게 한다.
- 전송 후 결과 확인이 미완료된 동작을 재조회까지 기억한다.
- 서버 반영이 확인되기 전에는 완료 이벤트를 만들지 않는다.

이 값들은 모두 프로세스 메모리에만 두고 지속 저장하지 않는다.

### 프레젠테이션

- 인증 카드와 출결 카드 중 하나만 렌더링한다.
- 카드 순서, 반응형 2×2 상태 타일, 날짜·시간 표시, 타일 강조와 스낵바를 담당한다.
- 완료 이벤트 식별값을 소비해 동일 이벤트의 중복 스낵바를 방지한다.
- 한국 날짜·요일 및 서버 시각 문자열 변환은 작은 표시 도우미로 분리한다.
- 기존 스냅샷을 유지한 채 재조회가 실패하면 출결 카드 안에 오류 메시지와
  `출결 상태 다시 조회` 동작을 표시한다.

### 도메인과 데이터

- `AttendanceSnapshot`의 원본 문자열과 `availableActions`, `reflects` 규칙은 유지한다.
- API 요청, 토큰 검증, 출결 전송 방식은 변경하지 않는다.

## 오류 처리

- 인증 시작 또는 콜백 실패: `출결 정보 확인하기` 카드 안에서 오류와 재시도를 표시한다.
- 상태 재조회 실패: 기존 오늘 출결 카드가 있다면 유지하고 오류·재조회 동작을 제공한다.
- 출결 반영 미확인: 성공 색상과 완료 스낵바를 표시하지 않는다.
- 시각 파싱 실패: 해당 원문을 표시하고 다른 출결 행은 정상 렌더링한다.
- 네트워크 불허: 평상시 네트워크 행은 숨기고 불허 경고만 표시한다.

## 검증

자동 테스트는 다음 동작을 확인한다.

- 인증 전 제목이 `출결 정보 확인하기`인지
- 인증 카드가 오늘 예정된 동작보다 위에 있는지
- 인증 성공 후 인증 카드 대신 같은 위치에 오늘 출결 카드 하나만 나타나는지
- 날짜와 한국어 요일이 제목에 표시되는지
- 일반 휴대폰 폭에서는 상태가 2×2로 배치되고 좁은 폭이나 큰 글자에서는 1열로
  안전하게 전환되는지
- ISO 8601과 `HH:mm:ss` 값이 `HH:mm`으로 변환되고 기존 `HH:mm` 값과 예외 문자열이
  안전하게 유지되는지
- `네트워크 허용: 예` 행이 사라지고 불허 경고는 유지되는지
- 단순 조회는 `방금 업데이트됨`만 표시하는지
- 서버 반영이 확인된 동작만 해당 타일 강조와 완료 이벤트를 만드는지
- 완료 스낵바가 같은 이벤트에 한 번만 표시되는지
- 전송 후 조회 실패에서는 완료 피드백이 없고 재조회에서 반영 확인 시 나타나는지

검증 명령은 `dart format`, `flutter analyze`, `flutter test`,
`flutter build apk --release --split-per-abi`를 사용한다. 최종 arm64 릴리스 APK는
기존 release 인증서, application ID와 증가한 배포 버전을 확인한 뒤 연결된 Android
실기기에 업데이트 설치한다. 실제 출결 동작은 검증을 위해 임의로 전송하지 않는다.

## 구현 순서

1. 표시 형식과 완료 이벤트에 대한 실패 테스트를 추가한다.
2. 날짜·시간 표시 도우미를 구현한다.
3. 컨트롤러에 조회 및 완료 이벤트 상태를 추가한다.
4. 인증 카드와 오늘 출결 카드의 교체 구조를 구현한다.
5. 행 강조, 업데이트 표시, 스낵바를 구현한다.
6. 전체 자동 테스트와 Android 릴리스 빌드를 실행한다.
7. 연결된 실기기에 업데이트 설치해 기존 데이터·권한 유지와 화면 배치를 확인한다.
