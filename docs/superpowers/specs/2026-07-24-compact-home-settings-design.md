# 홈 화면 압축과 통합 설정 화면 설계

## 목적

`오늘 출결` 카드가 항상 표시되면서 홈 화면의 `오늘 예정된 동작`이 첫 화면 아래로
완전히 밀렸다. 홈 화면은 매일 확인하는 출결 상태와 일정에 집중하고, 자주 바꾸지 않는
사용자 정보·테마·권한·앱 정보는 별도 설정 화면으로 이동한다.

기본 글자 크기의 일반 스마트폰에서는 별도 안내 없이도 아래로 스크롤할 내용이 있다는
것을 알 수 있도록, 첫 화면 하단에 `오늘 예정된 동작` 카드의 상단 또는 제목 일부가
보여야 한다. 접근성 글자 크기가 커졌을 때는 이 기준보다 내용의 비잘림과 조작 가능성을
우선한다.

## 홈 화면 정보 구조

### 제거

- `${이름}님, 안녕하세요`
- `오늘의 출결 일정과 상태를 확인하세요.`
- 카드형 사용자 정보와 `사용자 정보 변경` 버튼
- AppBar의 테마 전용 버튼
- 출결 카드 안의 별도 `Google 인증 다시 시도` 또는 상태 재시도 버튼
- 성공한 조회를 계속 차지하는 박스형 안내 문구

### 유지 및 변경

- AppBar 제목 `SKALA 출결`
- AppBar 우측에는 테마 버튼 대신 `설정` 톱니바퀴 버튼을 표시한다.
- 본문은 `오늘 출결` 카드로 바로 시작한다.
- `오늘 예정된 동작`과 `일정 관리`는 현재 순서와 기능을 유지한다.
- 출결 상태 타일은 2×2 구조와 완료 강조를 유지하되 세로 높이와 내부 여백을 줄인다.
- 카드 외부 간격과 `오늘 출결` 카드 내부 여백도 함께 줄인다.
- 큰 글자 또는 좁은 폭에서는 현재처럼 상태 타일을 한 열로 전환한다.

권장 기본 치수는 다음과 같다. 구현 중 실제 위젯 테스트와 실기기 화면을 기준으로
소폭 조정할 수 있지만, 터치 영역과 접근성은 줄이지 않는다.

- 출결 카드 내부 여백: 18px에서 약 14px
- 상태 타일 최소 높이: 86px에서 약 68~72px
- 상태 타일 내부 여백: 12px에서 약 10px
- 상태 타일 사이 간격: 8px 유지

## 출결 안내와 오류 복구

출결 카드 우측 상단 새로고침을 상태 조회와 인증 재시도의 단일 진입점으로 사용한다.
별도 재시도 버튼을 추가하지 않는다.

- 아직 조회하지 않은 상태:
  `우측 상단 새로고침 버튼을 눌러 Google 인증 후 출결 정보를 갱신하세요.`
- Google 인증을 취소했거나 완료하지 않은 상태:
  `Google 인증이 완료되지 않았습니다. 새로고침 버튼을 눌러 다시 시도해 주세요.`
- 그 밖의 조회 오류:
  원인을 설명한 기존 사용자용 문구 뒤에
  `새로고침 버튼을 눌러 다시 확인해 주세요.`를 붙인다.
- 조회 성공:
  박스형 성공 문구를 계속 표시하지 않는다. 기존 `방금 업데이트됨`, 타일 강조,
  완료 스낵바를 사용한다.

새로고침의 안전 규칙은 유지한다. 미확인 출결 전송이 있으면 새로고침은 상태만
대조하며 동작을 재전송하지 않는다.

## 설정 진입과 화면 구조

AppBar의 설정 버튼을 누르면 별도 전체 화면 `설정`으로 이동한다. 팝업이나 바텀시트가
아닌 화면으로 만들어 향후 항목이 늘어도 구조를 유지한다.

### 사용자 정보

- 이름
- 캠퍼스와 반
- `사용자 정보 변경`

행을 누르면 기존 사용자 정보 변경 화면을 연다. 검증에 성공한 정보만 저장하며,
변경 후 홈과 출결 컨트롤러에는 기존 방식대로 새 프로필이 반영된다.

### 화면 설정

- 현재 테마를 `시스템 설정`, `라이트`, `다크` 중 하나로 표시한다.
- 행을 누르면 기존 테마 선택 UI를 열고 선택 즉시 앱 전체에 반영·저장한다.

### 권한 및 필수 설정

Android에서는 다음 세 항목을 각각 표시한다.

1. 알림 권한
2. 정확한 알람 권한
3. 인증 후 앱 복귀 설정

iOS 보존 구현에서는 알림 권한만 표시한다. Android 전용 정확한 알람과 App Link
설정을 iOS에서 지원하는 것처럼 표시하지 않는다.

각 행은 `허용됨`, `설정 필요`, `확인할 수 없음` 중 하나의 상태를 아이콘과 함께
표시한다. 설정 화면 진입 시 상태를 읽고, 시스템 설정에서 앱으로 돌아오면 다시 읽는다.

- 알림 권한 행: 앱의 알림 설정으로 이동
- 정확한 알람 권한 행: 정확한 알람 허용 설정으로 이동
- 인증 후 앱 복귀 설정 행: 기존 App Link 설정 화면으로 이동

행을 누르는 것만으로 권한을 임의 변경하지 않는다. 시스템 설정에서 사용자가 직접
변경한다. 필수 권한이 해제된 경우 초기 설정으로 돌려보내는 현재 앱 전역 규칙은
유지하며, 설정 화면이 이를 우회하지 않는다.

### 앱 정보

- 표시 버전
- 빌드 번호

값은 실행 중인 설치 패키지의 메타데이터에서 읽는다. 문구나 소스 코드에
`1.0.0` 또는 `4010`을 하드코딩하지 않는다. 기본 표시는
`버전 1.0.0 (빌드 4010)` 형식으로 한다.

## 애플리케이션 구조

### 화면 경계

- `AttendanceScreen`은 더 이상 프로필 편집과 테마 선택 UI를 직접 소유하지 않는다.
- `AttendanceScreen`에는 `onOpenSettings` 콜백만 전달한다.
- 앱 조합 계층이 설정 화면을 열고 사용자 정보·테마 변경 콜백과 플랫폼 서비스를
  주입한다.
- 새 설정 화면은 `lib/features/settings` 아래의 application/data/presentation
  경계를 따른다.

### 권한 상태

프레젠테이션 위젯에서 플러그인이나 MethodChannel을 직접 호출하지 않는다.

- 알림 관련 서비스는 일반 알림과 정확한 알람 상태를 분리한 상태 모델을 제공한다.
- 기존 `arePermissionsGranted()`는 두 Android 권한을 모두 요구하는 현재 의미를
  유지하되, 내부적으로 분리된 상태 모델을 사용할 수 있다.
- Android 시스템 설정 이동은 data/platform 계층에서 담당한다.
- 인증 복귀 설정은 기존 `CallbackLinkSettings`를 재사용한다.
- 설정 화면 컨트롤러는 세 상태 조회, 로딩, 복귀 후 갱신, 사용자용 오류 문구만
  조율한다.

### 앱 버전

앱 버전 조회는 교체 가능한 제공자 인터페이스 뒤에 둔다. 실제 구현은 설치 패키지
메타데이터를 읽고, 테스트는 가짜 버전 값을 주입한다.

## 오류 처리

- 권한 상태 하나를 읽지 못해도 설정 화면 전체를 막지 않는다.
- 실패한 행만 `확인할 수 없음`으로 표시하고 다시 화면에 들어오거나 앱이 재개될 때
  재조회한다.
- 시스템 설정을 열지 못하면 설정 화면에 스낵바로 알린다.
- 버전 정보를 읽지 못하면 `버전 정보를 확인할 수 없음`으로 표시한다.
- 사용자 정보 변경 실패와 테마 저장 실패는 기존 처리 방식을 유지한다.

## 접근성

- 설정 버튼에 `설정` 툴팁과 의미 레이블을 제공한다.
- 새로고침 버튼의 기존 `출결 상태 새로고침` 레이블을 유지한다.
- 권한 상태를 색상만으로 구분하지 않고 텍스트와 아이콘을 함께 사용한다.
- 설정의 각 행은 Material 최소 터치 영역을 유지한다.
- 타일 높이를 줄이더라도 큰 글자에서는 고정 높이를 강제하지 않고 내용에 맞게
  확장하거나 한 열로 전환한다.

## 검증 기준

### 홈

- 인사말·설명·사용자 정보 카드가 표시되지 않는다.
- 설정 버튼이 홈 AppBar에 표시되고 테마 전용 버튼은 없다.
- 기본 스마트폰 크기와 기본 글자 크기에서 첫 화면 하단에
  `오늘 예정된 동작` 카드 상단 또는 제목 일부가 보인다.
- 출결 상태 타일은 기본 2×2, 큰 글자/좁은 폭에서는 한 열로 표시된다.
- 조회 전 안내와 인증 취소 안내가 새로고침 버튼을 명시한다.
- 별도 Google 인증 재시도 버튼이 없다.
- 성공 상태는 큰 안내 상자를 계속 차지하지 않는다.

### 설정

- 사용자 정보가 표시되고 기존 변경 화면을 열 수 있다.
- 현재 테마가 표시되고 변경이 즉시 홈에 반영된다.
- Android에서 알림·정확한 알람·인증 복귀 상태가 분리되어 표시된다.
- 각 권한 행이 올바른 시스템 설정을 연다.
- 시스템 설정 복귀 후 상태가 갱신된다.
- iOS에서는 Android 전용 권한 행을 표시하지 않는다.
- 설치 패키지에서 읽은 버전과 빌드 번호를 표시한다.

### 회귀

- 필수 권한 해제 시 초기 설정으로 복귀하는 기존 동작을 유지한다.
- 출결 새로고침, 동작 확인, 예약 알림, App Link 콜백 흐름을 변경하지 않는다.
- 실제 출결 동작을 테스트 목적으로 전송하지 않는다.

## 범위 밖

- 알림음·진동·반복 횟수를 편집하는 네이티브 알람 설정
- 권한 정책 완화 또는 필수 설정 건너뛰기
- iOS Universal Link 출결 콜백
- 일정 관리 화면의 구조 변경
- 홈 카드 순서 변경
