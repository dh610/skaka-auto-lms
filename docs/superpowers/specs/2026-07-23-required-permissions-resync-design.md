# 필수 설정 복구와 알림 재예약 설계

## 배경

Android 앱의 핵심 기능은 사용자가 지정한 시각에 출결 알림을 표시하고,
알림을 누른 뒤 Google 인증에서 앱으로 복귀하는 것이다. 따라서 다음 세
설정은 모두 필수다.

- 일반 알림 권한
- 정확 알람 특별 접근 권한
- App Link의 지원되는 링크 열기

현재 앱은 세 설정 중 하나라도 해제되면 앱 복귀 또는 다음 실행에서 초기
설정 화면으로 이동한다. 이 정책은 유지한다.

문제는 사용자가 정확 알람 권한을 해제한 뒤 앱을 종료하지 않고 다시
허용했을 때 발생한다. Android는 권한이 해제될 때 기존 정확 알람을 취소할
수 있지만, 현재 앱은 권한 복구 후 상태만 갱신하고 전체 알림을 다시
예약하지 않는다. 그 결과 설정 완료 화면을 통과했는데 실제 정확 알림은
없는 상태가 될 수 있다.

## 목표

1. 세 필수 설정을 모두 완료하기 전에는 메인 화면에 진입하지 않는다.
2. 초기 설정 화면의 `나중에 설정` 동작을 제거한다.
3. 세 설정이 모두 준비된 시점마다 저장된 최신 일정으로 전체 알림을
   재예약한다.
4. 필수 설정이 해제된 것을 감지하면 저장된 초기 설정 완료 상태도
   해제한다.
5. 전체 재예약이 성공해야 초기 설정을 다시 완료 처리한다.
6. 재예약에 실패하면 초기 설정 화면에 남아서 오류를 알리고 재시도할 수
   있게 한다.

## 접근 방법 비교

### 1. 초기 설정 완료 직전에 항상 전체 재예약

세 설정이 모두 준비되면 일정 로딩을 기다리고 전체 재예약을 실행한다.
성공한 경우에만 초기 설정 완료 상태를 저장한다.

- 장점: 최초 설정, 권한 재허용, App Link 복구를 하나의 완료 경로로 처리한다.
- 장점: 화면에 진입하기 전에 실제 알림 등록 성공까지 보장한다.
- 단점: App Link만 복구한 경우에도 알림 전체 재예약이 한 번 실행된다.

### 2. 정확 알람 권한이 `꺼짐 → 켜짐`으로 바뀔 때만 재예약

초기 설정 화면에서 이전 권한 상태를 기억하고 정확 알람 권한 전이만
감지한다.

- 장점: 불필요한 재예약 호출이 적다.
- 단점: 권한 상태 추적이 화면에 결합되고 최초 설정과 복구 경로가 나뉜다.
- 단점: 앱 외부에서 변경된 상태나 다른 필수 설정 복구를 놓치기 쉽다.

### 3. 앱 생명주기 전체에서 권한 전이를 추적

앱 최상위 상태가 모든 권한 전이를 감시하고 필요한 경우 재예약한다.

- 장점: 가장 일반적인 권한 복구 구조다.
- 단점: 현재 문제에 비해 상태 관리 범위가 크고 복잡하다.
- 단점: 초기 설정 화면의 완료 처리와 별도 동기화가 필요하다.

## 선택한 설계

1번 방식을 사용한다. 알림은 최대 60건이고 초기 설정 완료 시점에만 추가
재예약하므로, 불필요한 호출 한 번보다 완료 상태와 실제 OS 알림 상태가
일치한다는 보장이 더 중요하다.

## 동작 흐름

### 정상 완료

```text
초기 설정 화면
→ 일반 알림 권한 확인
→ 정확 알람 권한 확인
→ App Link 확인
→ 저장된 일정 로딩 완료 대기
→ 최신 일정으로 전체 알림 재예약
→ 재예약 성공
→ 초기 설정 완료 저장
→ 메인 화면 진입
```

### 권한 해제 후 복구

```text
사용자가 정확 알람 권한 해제
→ 앱 복귀 시 설정 미완료 감지
→ 저장된 초기 설정 완료 상태 해제
→ 초기 설정 화면 표시
→ 사용자가 정확 알람 권한 다시 허용
→ 세 필수 설정 재확인
→ 최신 일정으로 전체 알림 재예약
→ 성공 후 메인 화면 복귀
```

### 재예약 실패

```text
세 필수 설정 확인 완료
→ 전체 알림 재예약 실패
→ 초기 설정 완료 상태를 저장하지 않음
→ 초기 설정 화면 유지
→ 알림 예약 실패 메시지와 재시도 동작 제공
```

## 상태와 책임

- `InitialSetupScreen`은 세 필수 설정의 준비 여부와 완료 시도 중 UI를
  관리한다.
- 앱 최상위 상태는 일정 초기 로딩 완료를 기다린 뒤 알림 전체 재예약을
  요청하고, 성공한 경우에만 초기 설정 완료를 저장한다.
- 앱 최상위 상태는 시작 또는 앱 복귀 시 필수 설정이 하나라도 해제된 것을
  확인하면 저장소의 초기 설정 완료 상태도 해제한다. 따라서 재예약 실패 후
  앱을 재시작해도 메인 화면으로 우회하지 않는다.
- `ScheduleController`는 기존 단일 실행·최신 상태 합치기 규칙을 유지하면서
  호출자에게 이번 전체 재예약의 성공 여부를 전달한다.
- OS 알림 예약 실패는 일정 저장 데이터를 되돌리지 않는다.

## 오류 표시

재예약 실패 시 사용자가 원인을 해결하거나 다시 시도할 수 있도록 초기
설정 화면에 다음 의미의 메시지를 표시한다.

```text
알림을 예약하지 못했습니다. 설정을 확인한 뒤 다시 시도해 주세요.
```

세 권한 중 하나가 다시 꺼졌다면 먼저 해당 설정을 다시 안내한다. 세 권한이
모두 준비되어 있다면 같은 완료 버튼으로 전체 재예약을 다시 시도한다.

## 테스트 기준

- `나중에 설정` 버튼과 관련 안내 문구가 표시되지 않는다.
- 세 설정 중 하나라도 준비되지 않으면 초기 설정을 완료하지 않는다.
- 세 설정이 모두 준비되면 전체 재예약을 한 번 실행한다.
- 일정 로딩이 끝나기 전에 완료를 시도해도 로딩 이후 최신 일정으로
  재예약한다.
- 필수 설정이 해제되면 저장된 완료 상태도 해제된다.
- 재예약 성공 전에는 완료 상태를 저장하지 않는다.
- 재예약 실패 후 앱을 재시작해도 초기 설정 화면이 유지된다.
- 재예약 실패 시 초기 설정 화면과 오류 메시지가 유지된다.
- 실패 후 재시도에 성공하면 완료 상태를 저장하고 메인 화면으로 이동한다.
- 기존 재예약 직렬화와 알림 탭 유효성 검증 테스트가 계속 통과한다.

## 범위 밖

- 정확 알람 권한을 선택 권한으로 낮추는 변경
- 부정확 알람을 정상 운영 모드로 제공하는 변경
- 60개 알림을 백그라운드에서 순환 보충하는 기능
- App Link 서버 검증 방식 변경
