# 현재 구현 상태

## 문서 목적

이 문서는 2026-07-16 기준 애플리케이션에 실제로 구현된 범위와 아직 구현하지 않은
범위를 설명한다. 개발 초기의 가설과 조사 과정은
[초기 조사 및 구현 계획](implementation-notes.md)에 별도로 보존한다.

## 제품 동작 원칙

이 앱은 화면 잠금 상태에서 Google 인증과 출결 요청을 완전히 무인 실행하는 도구가
아니다. 사용자가 만든 일정에 맞춰 로컬 알림을 표시하고, 알림을 누르면 Google 인증을
시작한다. 인증 완료 후 서버의 현재 출결 상태를 확인한 다음 가능한 동작만 처리한다.

```text
사용자 일정 저장
  → OS 로컬 알림 예약
  → 사용자가 알림 선택
  → 외부 브라우저에서 Google 인증
  → Android App Link로 앱 복귀
  → 당일 상태 조회
  → 동작별 확인 정책 적용
  → 출결 요청
  → 상태 재조회로 반영 확인
```

## 구현된 기능

### 사용자 설정

- 최초 실행 시 이름, 캠퍼스 및 반 입력
- 판교캠퍼스 4F는 1~5반, 5F는 6~10반으로 선택 제한
- 사용자 정보 로컬 저장 및 수정
- 사용자 정보가 바뀌면 메모리의 인증 토큰과 조회 상태 초기화

### 일정과 공휴일

- 입실, 퇴실, 외출, 복귀 일정 지원
- 요일 반복 일정과 특정 날짜 1회 일정 지원
- 일정 생성, 수정, 활성화, 비활성화 및 삭제
- 과정 기간 `2026-07-14`~`2026-12-18` 적용
- 요일 반복 일정에서 2026년 과정 기간 공휴일 제외
- 현재 선택한 요일과 겹쳐 실제로 제외되는 공휴일 표시
- 날짜 지정 일정은 공휴일에도 유지

공휴일 데이터 선택 근거는 [공휴일 데이터 운영 전략](holiday-data-strategy.md)에
기록되어 있다.

### 알림

- Android와 iOS 로컬 알림 권한 요청
- Android 정확 알람 권한이 있으면 정확 알람 사용
- 권한이 없으면 Android에서 부정확 알람 방식으로 대체
- 앱 시작, 일정 저장·수정·삭제·활성화 변경 시 향후 알림 재계획
- 앱 재시작과 알림 탭으로 시작된 경우 모두 payload 처리
- 플랫폼 예약 한도를 고려해 가까운 일정부터 최대 60건 계획

알림에는 일정 ID와 출결 동작만 들어가며 인증 토큰이나 개인정보는 넣지 않는다.

### 인증과 Android 출결 처리

- 사용자 정보로 SKALA 사전 확인 API 호출
- 외부 Chrome에서 Google OAuth 시작
- `att.skala-ai.com` Android App Link 콜백 수신
- JWT 형식, 만료 시각, 한국 날짜, 지역 및 반 일치 여부 검증
- 토큰을 디스크에 저장하지 않고 앱 프로세스 메모리에서만 사용
- Bearer 토큰으로 당일 출결 상태 조회
- 현재 서버 상태에 따라 허용되는 동작만 노출 및 전송
- 출결 요청 후 상태를 재조회해 반영 여부 확인

현재 동작 가능 조건은 다음과 같다.

| 현재 상태 | 가능한 동작 |
|---|---|
| 입실 전 | 입실 |
| 입실 후, 외출 전 | 외출 또는 퇴실 |
| 외출 후, 복귀 전 | 복귀 |
| 복귀 후 | 퇴실 |
| 퇴실 완료 또는 네트워크 불허 | 없음 |

### 출결 확인 정책

- 입실: 확인창 없이 전송
- 복귀: 확인창 없이 전송
- 외출: 하루 한 번만 가능하다는 경고 후 사용자 확인 시 전송
- 17:50 이전 퇴실: 17:50까지 남은 시·분·초를 실시간으로 보여준 뒤 사용자 확인 시 전송
- 17:50 이후 퇴실: 확인창 없이 전송

확인창의 카운트다운은 확인창이 열린 동안에만 1초 간격으로 갱신된다. 17:50이 되어도
자동으로 퇴실하지 않으며 사용자가 전송 버튼을 눌러야 한다.

## 플랫폼별 현재 범위

| 기능 | Android | iOS |
|---|---|---|
| 프로필·일정·공휴일 계산 | 지원 | 지원 |
| 로컬 알림 | 지원 | 지원 코드 포함 |
| 외부 브라우저 인증 시작 | Chrome | Safari 등 시스템 브라우저 |
| 인증 후 앱 복귀 | 지원 | 미구성 |
| 앱 내 상태 조회·출결 전송 | 지원 | 미지원 |

자세한 원인과 iOS 확대 조건은
[Android와 iOS의 기능 차이](platform-capability-differences.md)를 참고한다.

## 저장 데이터와 보안

기기에 지속해서 저장하는 데이터:

- 이름, 캠퍼스, 반
- 사용자가 만든 일정과 활성화 상태

지속해서 저장하지 않는 데이터:

- Google 비밀번호
- Google 또는 브라우저 쿠키
- SKALA 출결 Bearer 토큰

공공 API 인증키 역시 앱에 포함하지 않는다. 인증 토큰과 개인정보 전문을 로그나 알림
payload에 기록하지 않는 것을 원칙으로 한다.

## 구현하지 않은 기능

- Google 계정 자동 선택 및 무인 OAuth
- 화면 잠금 상태에서 출결 API 자동 전송
- Wi-Fi 자동 연결·해제 또는 SSID 관리
- 출결 토큰의 영구 저장 및 다음 실행 재사용
- 백그라운드 재시도 Worker
- 날짜별 출결 성공·실패 이력 화면
- 임시공휴일의 원격 실시간 갱신
- iOS Universal Link 콜백과 앱 내 출결 처리

이 목록은 초기 계획에서 의도적으로 제외했거나 향후 검증이 필요한 범위다. 특히 인증과
백그라운드 동작은 플랫폼 보안 정책을 우회하는 방식으로 구현하지 않는다.

## 검증 현황

- Flutter 정적 분석 통과
- 모델, 일정, 공휴일, 알림 계획, API 및 주요 위젯 흐름 자동 테스트
- Android 실기기에서 외부 브라우저 인증, App Link 복귀, 상태 조회 검증
- Android 실기기에서 예약 알림 탭 후 인증 및 예약 동작 연결 검증
- iOS 실기기 검증은 아직 수행하지 않음

실제 출결 동작은 테스트 편의를 위해 임의로 전송하지 않고 올바른 출결 시점과 사용자
의사에 따라 검증한다.
