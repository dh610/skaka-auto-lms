# SKALA 출결 도우미

SKALA LMS의 Google 인증을 거쳐 당일 출결 상태를 확인하고 출결 동작을 처리하는
Flutter 앱입니다. 현재 배포 및 유지보수 대상은 Android이며, 앱 내 상태 조회와 출결
이벤트 전송을 지원합니다. iOS 구현은 기술 검증 결과를 보존하기 위해 저장소에 남겨
두지만 신규 기능 개발과 사용자 배포는 중단한 상태입니다.

## 지원 플랫폼

- **Android:** 현재 개발·검증·배포 대상
- **iOS:** 코드와 시뮬레이터 검증 결과만 보존하며 개발·배포 중단

## 현재 기능

- 최초 실행 시 이름, 캠퍼스, 반을 SKALA 사전 확인 API로 검증한 뒤 설정
- 저장된 사용자 정보를 조회하고, 변경 시에도 검증한 뒤 저장
- 시스템 설정, 라이트 모드, 다크 모드 중 앱 테마 선택 및 저장
- 출결 일정 생성, 수정, 활성화 및 삭제
- 일정 편집 중 고정 저장 버튼과 진입 시 스크롤 가능 여부 안내
- 제스처·3버튼 내비게이션의 시스템 영역을 반영한 저장 버튼 하단 여백
- 요일·시각·동작별 일정 로컬 저장
- 요일 반복 일정의 2026년 과정 기간 공휴일 제외
- 과정 기간 내 특정 날짜 1회 일정
- 실제 발생 날짜와 시각이 겹치는 중복 일정 저장 방지
- 메인 화면에서 오늘 알림 일정의 예정·완료·시간 지남·건너뜀 상태 확인
- 시간 지남 일정을 발생 건별로 건너뜀 처리하거나 되돌리기
- 일정 알림을 통해 인증·브라우저 흐름을 시작한 상태를 발생 건별로 당일 보존하고 자정 이후 초기화
- Android 전체 화면 알람과 보존된 iOS 구현의 로컬 일정 알림
- Android 일정별 알람음·음량·진동·점진적 음량·다시 알림 설정
- 최초 사용자 정보 저장 후 일반 알림·정확 알람·전체 화면 알람·Android 인증 복귀 설정을 필수로 확인하고 알림 전체 재예약이 성공하면 앱 사용 시작
- 공휴일을 제외한 향후 일정 알림을 가까운 순서대로 최대 60건 선등록
- 캠퍼스에 따른 반 선택 제한
  - 판교캠퍼스 4F: 1~5반
  - 판교캠퍼스 5F: 6~10반
- Android Chrome Custom Tab 및 보존된 iOS 앱 내 Safari 화면을 통한 Google OAuth 시작
- 날짜·요일이 표시되는 `오늘 출결` 카드를 항상 표시하고, 조회 전 네 상태를 `확인 전`으로 표시
- 카드 우측 상단 새로고침으로 출결 상태만 조회하며, 인증이 필요할 때만 Google 인증 시작
- 입실·퇴실·외출·복귀를 누르면 인증이 필요할 때 인증을 시작하고, 최신 상태 조회와 확인 절차 뒤에만 Android 출결 요청 전송
- 조회된 출결 시각은 `HH:mm`, 조회 결과에 없는 항목은 `없음`으로 표시
- Android App Link 콜백 수신 및 당일 출결 상태 조회. 같은 앱 실행 중 유효한 토큰은 재사용할 수 있지만 저장하지 않음
- 오늘의 한국 날짜, 네 출결 표시 시각, 조회 시각만 로컬에 보관하고 날짜나 사용자 정보가 바뀌면 초기화
- 서버 상태에서 반영이 확인된 출결 동작의 행 강조 및 완료 안내
- 인증·상태 조회 실패에 대한 원인별 안내 및 안전한 재시도
- 예약 알림에서 Google 인증 후 예정된 출결 동작 연결
- 외출 및 오후 5시 50분 이전 퇴실에 대한 확인 절차

## 주의

- Google 비밀번호는 앱에 저장하지 않습니다.
- 인증 토큰 전문을 로그나 저장소에 남기지 않습니다.
- iOS 코드는 삭제하지 않지만 현재 배포 대상이 아니며 실기기 지원을 보장하지 않습니다.
- iOS에서는 인증 후 앱 콜백과 앱 내 출결 요청을 지원하지 않습니다.
- iOS의 브라우저 인증 화면은 사용자가 SKALA 웹에서 직접 처리하는 용도이며, Android App Link 흐름을 대신하지 않습니다.
- 예약 알림은 무인 출결 실행이 아니라 사용자가 Google 인증을 시작하는 진입점입니다.
- Android에서 VPN이 활성화되면 SKALA 허용 네트워크 판별을 방해할 수 있으므로 필요하면 이 앱을 VPN 적용 대상에서 제외해야 합니다.
